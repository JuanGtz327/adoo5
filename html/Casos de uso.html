Especificación Casos de Uso

CU1. Registrar Producto
Autor: Gerardo
Versión: 1
Fecha de elaboración: 23 de abril
Revisor:
Estado: en elaboración
Actores: Administrador
Entradas: Información de producto, imágenes del producto.
Salidas: Página con una nueva entrada al catálogo.
Precondiciones: Tener iniciada la sesión en el sistema, que no exista un registro previo o duplicado del producto.
Postcondiciones: Nuevo producto registrado en el sistema con toda la información necesaria para que sea publicado.
Trayectoria principal:

  1. El caso de uso comienza cuando se quiere añadir un nuevo producto al sistema.
  2. El administrador selecciona la opción Registrar Nuevo Producto a través de la opción del IU1. Menú de Opciones del sistema.
  3. El administrador introduce los datos y las imágenes del producto a registrar a través de las opciones del IU2. Registrar Nuevo Producto.
  4. El sistema valida los datos introducidos de acuerdo a las RN1 y RN2 (Alternativa A)
  5. El sistema muestra que el producto fue registrado satisfactoriamente a través del IU3. Producto Registrado.

6.Fin del caso de uso.
Trayectorias alternativas:
Alternativa A: Los datos introducidos no son válidos.
1.El sistema muestra que el producto no fue registrado a través del IU4. Producto no registrado
2.El sistema regresa a la IU2. Registrar Nuevo Producto.
3.Fin del caso de uso

CU2. Registrar Venta
Autor: Gerardo
Versión: 1
Fecha de elaboración: 23 de abril
Revisor:
Estado: en elaboración
Actores: Sistema
Entradas: Información de producto vendido.
Salidas: Nueva entrada a la base de datos de ventas.
Precondiciones: Tener productos vendidos.
Postcondiciones: Nueva venta registrada en el sistema con la información de los productos vendidos para poder ser visualizada.
Trayectoria principal:
1.El caso de uso comienza cuando un producto fue vendido.
2.El sistema obtiene la información de los productos vendidos de acuerdo a la RN4. (cantidad, código del producto, nombre, tipo, categoría, total, imágenes y usuario al que fue vendido).
3.El sistema añade la información de la venta a la base de datos.
4.Fin del caso de uso.

CU3. Pagar Producto
Autor: Gerardo
Versión: 1
Fecha de elaboración: 23 de abril
Revisor:
Estado: en elaboración
Actores: Cliente
Entradas: Información de los productos a pagar, información bancaria del cliente.
Salidas: Pago hecho satisfactoriamente.
Precondiciones: Tener iniciada la sesión en el sistema y haber seleccionado productos a comprar.
Postcondiciones: Pago hecho para que la venta pueda ser registrada de acuerdo con CU2. Registrar Venta.
Trayectoria principal:

  1. El caso de uso comienza cuando se quiere pagar los productos seleccionados a comprar.
  2. El cliente selecciona la opción Pagar a través de la IU4. Menú de compra.
  3. El cliente introduce sus datos bancarios a través de las opciones del IU5. Información de pago.
  4. El sistema valida los datos introducidos de acuerdo a la RN4 (Alternativa A)
  5. El sistema de pagos valida los datos bancarios del cliente de acuerdo a la RN5 (Alternativa B)
  6. El sistema muestra que el pago fue válido a través de la IU6. Pago hecho.

13. Fin del caso de uso.
Trayectorias alternativas:
Alternativa A: Los datos introducidos no son válidos para el sistema.
1.El sistema muestra que los datos del cliente son inválidos a través del IU7. Datos inválidos.
2.El sistema regresa a la IU5. Información de pago.
3.Fin del caso de uso
Alternativa B: El sistema de pagos invalida la transacción.
1.El sistema muestra que el pago fue invalidado de acuerdo a los datos bancarios del cliente a través del IU8. Problema con el pago.
2.El sistema regresa a la IU5. Información de pago.
3.Fin del caso de uso

CU6. Responder comentarios
Autor: Gerardo
Versión: 1
Fecha de elaboración: 23 de abril
Revisor:
Estado: en elaboración
Actores: Administrador
Entradas: Mensaje enviado por el cliente.
Salidas: Mensaje enviado hacia el cliente.
Precondiciones: Tener iniciada la sesión en el sistema.
Postcondiciones: El cliente tendrá respuesta a su mensaje enviado.
Trayectoria principal:

  1. El caso de uso comienza cuando se quiere añadir un nuevo producto al sistema.
  2. El administrador selecciona la opción Responder feedback a través de la opción del IU1. Menú de Opciones del sistema.
  3. El administrador selecciona el mensaje que quiere responder mediante la IU12. Seleccionar mensaje del cliente.

4.El administrador introduce el mensaje que quiere enviar al cliente a través de la IU13. Responder mensaje del cliente.
5.El sistema envía la información del mensaje al cliente.
6.El sistema muestra que el mensaje fue enviado satisfactoriamente a través de la IU11. Mensaje enviado.
7.Fin del caso de uso.

CU7. Compartir información de producto en redes sociales.
Autor: Gerardo
Versión: 1
Fecha de elaboración: 23 de abril
Revisor:
Estado: en elaboración
Actores: Cliente
Entradas: Red social elegida por el cliente.
Salidas: Página de red social redirigida desde el sistema.
Precondiciones: Tener iniciada la sesión en el sistema y en la red social.
Postcondiciones: Producto promocionado en red social.
Trayectoria principal:

  1. El caso de uso comienza cuando se quiere compartir un producto a través de redes sociales.
  2. El cliente selecciona la opción Compartir a través de la opción del IU1. Menú de Opciones del sistema.
  3. El cliente selecciona la red social para compartir el producto a través de IU14. Redes sociales.

4.El sistema redirige hacia el sitio web de la red social seleccionada.
5.El cliente introduce el mensaje en la red social seleccionada.
6.El sistema redirige hacia la IU1. Menú de Opciones del Sistema.
7.Fin del caso de uso.
